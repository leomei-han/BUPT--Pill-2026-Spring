# 项目结构说明

## 1. 项目概述

本项目是一个基于 Verilog 的药片装瓶控制系统，采用自顶向下的模块化结构实现。
目标器件为 Altera EPM7128SLC84-15（MAX7000S 系列 CPLD）。

当前已经实现的功能包括：

- 药片逐颗计数（1 Hz 脉冲模拟药片传感器）
- 自动装瓶与满瓶换瓶
- 参数可配置（每瓶装入量 01–50，目标瓶数 01–18）
- 4 状态有限状态机顺序控制
- 两级报警机制（完成提醒 + 持续警报）
- 瓶满预测检测（提前一拍判断，零延迟换瓶）
- 按键消抖与边沿检测
- BCD 输入合法性校验
- 安全联锁（工作中无法切回配置态）
- 跨时钟域同步
- 上电默认参数
- 配置项闪烁指示
- 达标后数码管整体闪灭
- 警报态超量药片继续计数

整个工程采用双时钟域设计：

- 主逻辑使用 10 kHz 时钟驱动
- 1 Hz 慢时钟通过同步链采样后转换为单周期脉冲
- 所有寄存器更新均在 10 kHz 时钟域完成

这种写法避免了跨时钟域亚稳态问题，适合 CPLD 综合。

## 3. 顶层结构

主顶层模块为 `pill_ctrl_top.v`。

顶层本身不承担复杂计算功能，主要职责是：

- 将开关分组映射为语义信号
- 对 QD 按键进行消抖处理，并同步普通复位电平开关
- 锁存用户设置的参数
- 实例化并互联各个子模块

顶层输入输出如下：

### 输入

- `clk_10kHz`
  - 10 kHz 主时钟信号
- `clk_1Hz`
  - 1 Hz 药片节拍时钟信号
- `QD`
  - 脉冲按钮（确认输入、启动工作、继续操作）
- `reset_sw`
  - TEC-8 K15 普通电平复位开关（PIN_63，高电平有效，先同步后作为系统复位）
- `switches[10:1]`
  - 10 位拨码开关

### 输出

- `display1[4:1]` ~ `display5[4:1]`
  - 5 位数码管 BCD 码输出
- `buzzer`
  - 蜂鸣器控制信号


## 4. 当前硬件假设

根据当前约定，本工程默认硬件条件如下：

- 5 位 BCD 数码管（每位 4-bit 并行输出）
- 10 位拨码开关（SW1–SW8 数据，SW9 设置选择，SW10 模式选择）
- QD 脉冲按钮（高电平有效）
- 普通电平复位开关 `reset_sw`（TEC-8 K15 / PIN_63，高电平有效，顶层同步后使用）
- 蜂鸣器高电平有效
- 器件：EPM7128SLC84-15

如果实际板卡与上述不同，需要修改顶层的极性处理和 `.qsf` 引脚约束。


## 5. 系统总数据流

完整的控制/数据路径如下：

1. `pulse_sync` 用 10 kHz 采样 1 Hz，产生药片节拍脉冲
2. `pill_ctrl_top` 对 QD 按键消抖并检测边沿，对 `reset_sw` 做同步
3. `sw_checker` 实时检查拨码开关输入是否为合法 BCD 范围
4. `pill_ctrl_top` 在配置态下由 QD 确认后锁存参数
5. `bottle_fsm` 根据状态机驱动计数、换瓶、报警逻辑
6. `seg_output` 根据当前状态选择显示内容并控制闪烁
7. 5 路 BCD 码直接输出到数码管硬件

这种结构的优点是模块职责清晰：

- 信号调理归 `pill_ctrl_top`
- 节拍生成归 `pulse_sync`
- 输入校验归 `sw_checker`
- 流程控制归 `bottle_fsm`
- 显示逻辑归 `seg_output`


## 6. 各模块详细说明

## 6.1 `pill_ctrl_top.v`

### 作用

系统顶层模块，完成信号互联与预处理。

### 主要功能

- 开关映射：将 10-bit 开关分组为模式选择、设置选择、数据输入
- QD 消抖：2-bit 移位寄存器检测上升沿，输出单周期脉冲
- reset_sw 同步：2-bit 同步链将普通复位电平开关转为同步复位
- 参数锁存：配置态下 QD 确认时，根据设置选择开关分别锁存装入量或瓶数，非法输入锁存默认值
- 默认参数：复位后自动恢复为每瓶 10 颗、共 5 瓶

### 设计特点

按键消抖采用移位寄存器方案而非计数器方案。

QD 消抖写法为：

```verilog
always @(posedge clk_10kHz)
    r_qd <= {r_qd[0], QD};

assign trig_pulse = (r_qd == 2'b01);
```

reset_sw 使用 2 级同步链，高电平有效，先同步后作为系统复位：

```verilog
always @(posedge clk_10kHz)
    r_reset_sw <= {r_reset_sw[0], raw_reset_sw};

assign sys_rst = r_reset_sw[1];
```

参数锁存在配置模式且 QD 按下时写入：输入合法时锁存拨码值，输入非法或越界时锁存默认值。


## 6.2 `pulse_sync.v`

### 作用

用快时钟采样慢时钟，产生单周期有效脉冲。

### 输入

- `clk_10kHz`：10 kHz 主时钟
- `clk_1Hz`：1 Hz 外部慢时钟
- `reset`：同步复位

### 输出

- `pill_pulse`：每个 1 Hz 上升沿对应的单拍脉冲

### 内部实现方式

- 3-bit 移位寄存器对 1 Hz 信号连续采样 3 拍
- 取中间两位 `sr[2:1]` 做上升沿判断：`2'b01` 表示上升沿

### 设计特点

输出的不是 1 Hz 时钟，而是"单拍有效脉冲"。

后续模块的写法都是：

```verilog
always @(posedge clk) begin
    if (pill_pulse) begin
        ...
    end
end
```

而不是：

```verilog
always @(posedge clk_1Hz) begin
    ...
end
```

这避免了在 CPLD 上引入第二个时钟域。


## 6.3 `sw_checker.v`

### 作用

对拨码开关输入进行实时合法性验证。

### 输入

- `input_data[7:0]`：两位 BCD 数据
- `setting_switch`：当前正在配置哪个参数（0 = 装入量，1 = 瓶数）

### 输出

- `input_valid`：输入合法标志

### 验证规则

装入量模式（`setting_switch = 0`）：
- 十位 0–4：直接合法（对应 00–49）
- 十位 5、个位 0：合法（对应 50）
- 其余：非法

瓶数模式（`setting_switch = 1`）：
- 十位 0：直接合法（对应 00–09）
- 十位 1、个位 0–8：合法（对应 10–18）
- 其余：非法

### 设计特点

本模块为纯组合逻辑，无寄存器。
先检查两位是否均为合法 BCD（≤ 9），再做范围判断。
输出实时跟随输入变化，无需时钟。


## 6.4 `bottle_fsm.v`

### 作用

装瓶流程的核心状态机，负责计数、换瓶和报警控制。

### 状态定义

- `S_CFG (00)`：参数配置态，计数器持续清零
- `S_RUN (01)`：正常运行态，逐颗计数
- `S_HALT (10)`：目标达成提醒，蜂鸣器响起
- `S_WARN (11)`：持续警报态，仅复位可退出

### 状态切换规则

- `S_CFG → S_RUN`：QD 按下且模式开关置工作
- `S_RUN → S_HALT`：满瓶且瓶数达到目标
- `S_HALT → S_WARN`：QD 按下确认
- `S_WARN → S_CFG`：仅通过 reset_sw 复位
- 任意 `default → S_CFG`：非法状态兜底

### 最重要的设计原则

安全联锁：运行态下完全不检查模式开关的变化。

即使用户在工作中把 SW10 拨回配置位，系统也不会中断生产流程。
要回到配置态，必须通过 reset_sw 复位。

这防止了误操作导致计数丢失。

### BCD 计数实现

内部定义了两个 BCD 加一函数：

- `inc12`：12-bit BCD（3 位十进制，000–999），用于药片累计
- `inc8`：8-bit BCD（2 位十进制，00–99），用于本瓶药片数和瓶数

两个函数均采用逐位进位方式：个位满 9 则归零并向十位进位，以此类推。
溢出时回绕到 000 / 00。

### 瓶满预测

```verilog
wire bottle_about_full = (acc_cur == (pills_per_bottle - 8'h01))
                         && (pills_per_bottle != 8'h00);
```

在当前药片数等于"设定值减一"时，提前判定下一颗将装满。
这样在 `pill_pulse` 到来的那一拍就能同时完成：计数 → 清零 → 换瓶 → 检查目标。
没有多余的延迟周期。

### 蜂鸣器控制

```verilog
assign buzzer = (work_state == S_HALT) | (work_state == S_WARN);
```

纯组合逻辑，提醒态和警报态都驱动蜂鸣器。

### 警报态特殊行为

进入 `S_WARN` 后系统仍然接收 `pill_pulse` 并继续累计。
这是为了防止传送带上残余药片的数据丢失。
换瓶检测在警报态同样生效。


## 6.5 `seg_output.v`

### 作用

根据系统状态决定 5 位数码管的显示内容，并实现闪烁提示。

### 显示页面

配置画面（`mode_switch = 0`）：
- 位 1：固定显示 0
- 位 2–3：目标瓶数
- 位 4–5：每瓶装入量

运行画面（`mode_switch = 1`）：
- 位 1–3：已装药片总数（百-十-个）
- 位 4–5：当前瓶内药片数

### 闪烁规则

- 配置态 + 正在编辑装入量：位 4–5 以 1 Hz 闪烁
- 配置态 + 正在编辑瓶数：位 2–3 以 1 Hz 闪烁
- 达标提醒态：全部 5 位以 1 Hz 闪灭

闪灭时输出 `4'hF`（数码管显示空白）。

### 输入预览

在真实配置态下，如果正在编辑某个参数且当前输入合法，
数码管实时显示拨码开关的值（而非已锁存值），让用户预览即将设置的参数。
如果输入非法或越界，被编辑参数位显示确认时将采用的默认值并闪烁；
`display1` 在 `4'h0` 与 `4'hF` 之间闪烁，用作修正提示位。

### 输出说明

当前实现为了节省 EPM7128 的宏资源，显示值由组合逻辑直接生成。
如需增加输出寄存器去毛刺，会额外消耗约 20 个显示寄存器，不适合当前 128 macrocells 约束。

```verilog
always @(*) begin
    display1 = ...;
    ...
end
```


## 7. 开关和按键映射

系统共使用 10 个拨码开关和 2 个按钮：

- `switches[10]` (SW10)
  - 模式选择
  - `1` 表示工作模式
  - `0` 表示配置模式
- `switches[9]` (SW9)
  - 设置选择
  - `0` 表示配置每瓶装入量
  - `1` 表示配置目标瓶数
- `switches[8:1]` (SW8–SW1)
  - 两位 BCD 数据输入
  - 高 4 位为十位，低 4 位为个位
- `QD`
  - 脉冲按钮
  - 配置态：确认当前参数
  - 配置态 + SW10=1：启动运行
  - 提醒态：确认完成
- `reset_sw`
  - TEC-8 K15 普通电平复位开关（PIN_63，高电平有效）
  - 任意状态下可复位回配置态


## 8. 显示路径说明

本项目采用 BCD 并行输出方式，而非动态扫描。

显示路径如下：

1. `seg_output` 根据当前状态生成 5 组 4-bit BCD 值
2. 经过一级输出寄存器消除毛刺
3. 5 路 BCD 码直接连接到数码管的译码驱动硬件

优点是：

- 不需要扫描时钟
- 5 位数码管同时稳定显示
- 软件复杂度低
- 适合 IO 资源充裕的 CPLD


## 9. 安全机制说明

### 按键消抖

QD 使用 2-bit 移位寄存器检测上升沿，消除机械抖动。
reset_sw 使用 2-bit 同步链进入 `clk_10kHz` 时钟域后生成系统复位。

### 输入合法性校验

非法 BCD 值（如 `0xAB`）和越界值（如装入量 = 60）均被拒绝。
配置态按 QD 确认时，如果校验不通过，系统锁存对应默认值：
装入量默认 10，目标瓶数默认 05，并通过数码管闪烁提示。

### 安全联锁

工作状态下模式开关的变化被完全忽略。
只有 reset_sw 复位可以退出工作态，防止误拨开关导致生产参数丢失。

### 复位安全

reset_sw 触发后：
- 状态机回到配置态
- 所有计数器清零
- 参数恢复为默认值（10 颗/瓶、5 瓶）
- 蜂鸣器关闭

保证任何故障场景下都能通过复位恢复到已知安全状态。


## 10. 项目优点

当前结构适合作为课设实现，主要优点有：

- 模块划分清晰，5 个文件各司其职
- 状态机设计严格，无非法悬空状态
- 安全联锁防止误操作
- 预测性检测实现零延迟换瓶
- BCD 直接存储，省去二进制转换
- 跨时钟域处理规范
- 组合输出加寄存器去毛刺


## 11. 当前限制

这些属于当前范围限制，不是 bug：

- reset_sw 复位后计数进度不保留（不支持断点续传）
- 无超时故障检测（传感器卡住不会报警）
- 警报态超量计数在极端情况下可能 BCD 溢出（回绕至 000）
- 不支持运行中动态修改参数
- 无硬件仿真 testbench

## 14. 总结

本项目的核心结构可以概括为：

- `pulse_sync` 产生药片节拍
- `pill_ctrl_top` 完成信号预处理与参数管理
- `sw_checker` 保障输入合法性
- `bottle_fsm` 驱动装瓶流程状态机

在当前实现范围内，这套结构是清晰、稳定、可维护的，覆盖了课程设计要求的全部基本功能和加分项。
