# 项目结构说明

## 1. 项目概述

本项目是一个基于 Verilog HDL 的药片装瓶控制系统，目标器件为 Altera EPM7128SLC84-15（MAX7000S 系列 CPLD），使用 Quartus II 9.0 SP2 综合与布局布线。

系统采用双时钟域设计：

- **10 kHz 主时钟**：驱动所有时序逻辑（消抖、FSM、计数、参数锁存）
- **1 Hz 慢时钟**：经 `pulse_sync` 同步后产生单周期药片节拍脉冲

所有寄存器更新均在 10 kHz 时钟域完成，避免跨时钟域亚稳态。

## 2. 文件组成

### Verilog 源文件

| 文件 | 模块 | 职责 |
|------|------|------|
| `pill_ctrl_top.v` | `pill_ctrl_top` | 顶层模块：信号互联、开关映射、按键消抖、复位同步、参数锁存、蜂鸣器方波驱动 |
| `bottle_fsm.v` | `bottle_fsm` | 装瓶状态机：4 状态 FSM、3 组 BCD 计数器、换瓶/达标判定、蜂鸣器使能 |
| `seg_output.v` | `seg_output` | 显示驱动：5 位数码管内容选择、配置预览、闪烁控制、LG1 七段状态指示 |
| `pulse_sync.v` | `pulse_sync` | 脉冲同步：3 级移位寄存器跨时钟域同步 + 上升沿检测 |
| `sw_checker.v` | `sw_checker` | 输入校验：BCD 合法性与范围检查（纯组合逻辑） |

### Quartus 工程文件

| 文件 | 说明 |
|------|------|
| `pill_ctrl_top.qpf` | 工程描述文件，指定 Quartus 版本和工程修订 |
| `pill_ctrl_top.qsf` | 器件选型（MAX7000S）、引脚约束（37 个引脚）、综合设置 |

## 3. 模块层次

```
pill_ctrl_top (顶层)
  ├── pulse_sync        1 Hz → 10 kHz 域单周期脉冲
  ├── sw_checker         BCD 输入范围校验
  ├── bottle_fsm         装瓶 FSM + BCD 计数
  └── seg_output         数码管 + LG1 驱动
```

## 4. 顶层模块 `pill_ctrl_top.v`

### 端口

**输入：**

| 信号 | 宽度 | 说明 |
|------|------|------|
| `clk_10kHz` | 1 | 主时钟 |
| `clk_1Hz` | 1 | 药片节拍慢时钟 |
| `QD` | 1 | 确认/启动脉冲按键 |
| `reset_sw` | 1 | 复位开关（高电平有效） |
| `switches[10:1]` | 10 | 拨码开关 |

**输出：**

| 信号 | 宽度 | 说明 |
|------|------|------|
| `display1[4:1]` ~ `display5[4:1]` | 5×4 | 数码管 BCD 码 |
| `LG1[6:0]` | 7 | 七段状态指示 |
| `buzzer` | 1 | 蜂鸣器驱动 |

### 开关映射

```
switches[10]   → sw_mode    模式选择（0=配置，1=工作）
switches[9]    → sw_sel     参数选择（0=装入量，1=瓶数）
switches[8:1]  → sw_dat     8-bit BCD 数据
```

### 内部功能

**QD 消抖**：2-bit 移位寄存器在 10 kHz 下采样，检测 `2'b01` 模式捕获上升沿，输出单周期脉冲 `trig_pulse`。

**复位同步**：2 级同步链将 `reset_sw` 电平开关同步到 10 kHz 域，输出 `sys_rst`。

**参数锁存**：配置模式（`sw_mode=0`）下按 QD 时，根据 `sw_sel` 分别锁存装入量或瓶数。输入合法时锁存拨码值，非法时锁存默认值。复位时恢复默认值。

| 参数 | 默认值 |
|------|--------|
| 每瓶装入量 `cfg_fill` | `8'h10`（10 颗） |
| 目标瓶数 `cfg_goal` | `8'h05`（5 瓶） |

**蜂鸣器驱动**：2-bit 计数器分频 10 kHz 产生约 2.5 kHz 方波，驱动无源蜂鸣器。通过 `fsm_st[0] | clk_1Hz` 门控实现两种音效：
- HALT 态（`fsm_st = 2'b10`）：`fsm_st[0]=0`，仅 `clk_1Hz` 高电平期间输出 → 间歇鸣叫
- WARN 态（`fsm_st = 2'b11`）：`fsm_st[0]=1`，始终输出 → 持续鸣叫

## 5. 脉冲同步模块 `pulse_sync.v`

将 1 Hz 慢时钟同步到 10 kHz 主时钟域，输出单周期脉冲。

- 3-bit 移位寄存器连续采样 3 拍
- 取 `sr[2:1] == 2'b01` 判定上升沿
- 输出 `pill_pulse`：每秒一个单时钟周期有效脉冲

后续模块统一使用 `if (pill_pulse)` 在 10 kHz 域内处理，而非直接使用 `clk_1Hz` 作时钟，避免引入第二时钟域。

## 6. 输入校验模块 `sw_checker.v`

纯组合逻辑，实时校验 8-bit BCD 拨码输入的合法性。

**前置检查**：高低 4 位各自 ≤ 9（排除非法 BCD 编码）。

**范围检查**：

| 参数 | `setting_switch` | 合法范围 | 逻辑 |
|------|-------------------|----------|------|
| 每瓶装入量 | `0` | 01–50 | 十位 0~4 全通过（个位非零时），十位=5 仅允许个位=0 |
| 目标瓶数 | `1` | 01–18 | 十位=0 时个位非零，十位=1 时个位 ≤ 8 |

两种参数均排除 `00`（通过 `lo_nonzero` 检查）。

## 7. 状态机模块 `bottle_fsm.v`

### 状态定义

| 状态 | 编码 | 含义 |
|------|------|------|
| `S_CFG` | `2'b00` | 参数配置，计数器持续清零 |
| `S_RUN` | `2'b01` | 正常装瓶运行 |
| `S_HALT` | `2'b10` | 目标达成提醒 |
| `S_WARN` | `2'b11` | 持续警报（超量继续计数） |

### 状态转移

```
S_CFG ──(qd_pressed & mode_switch)──▶ S_RUN
S_RUN ──(满瓶 & 瓶数达标)──────────▶ S_HALT
S_HALT ──(qd_pressed)──────────────▶ S_WARN
任意 ──(reset)──────────────────────▶ S_CFG
default ────────────────────────────▶ S_CFG
```

### BCD 计数器

3 组 BCD 计数器：

| 寄存器 | 宽度 | 范围 | 用途 |
|--------|------|------|------|
| `acc_total` | 12-bit | 000–999 | 药片累计总数 |
| `acc_cur` | 8-bit | 00–99 | 当前瓶药片数 |
| `acc_btl` | 8-bit | 00–99 | 已完成瓶数 |

BCD 加一通过自定义函数 `inc12` 和 `inc8` 实现，采用等值检测（`== 4'd9`）判断进位，而非算术取模，节省 CPLD 乘积项资源。溢出时回绕至 0。

### 预判逻辑

```verilog
wire bottle_about_full = (acc_cur == (pills_per_bottle - 8'h01))
                         && (pills_per_bottle != 8'h00);
wire goal_about_done   = (acc_btl == (target_bottles - 8'h01))
                         && (target_bottles != 8'h00);
```

在计数到达设定值**前一个**时提前判定，使得下一个 `pill_pulse` 到来时能在同一拍内完成：计数 → 清零 → 换瓶 → 状态切换，实现零延迟换瓶。

### 蜂鸣器使能

```verilog
assign buzzer = work_state[1];
```

`S_HALT`（`10`）和 `S_WARN`（`11`）的 bit[1] 均为 1，统一触发蜂鸣器。实际音效区分由顶层门控逻辑完成。

### 警报态行为

`S_RUN` 和 `S_WARN` 共享计数逻辑分支（`case` 中 `S_RUN, S_WARN:`），药片在警报态仍然继续计数和换瓶。WARN 态无法自行退出，只能通过 reset。

## 8. 显示驱动模块 `seg_output.v`

### 显示内容

**配置态（`work_state == S_CFG`）：**

| display1 | display2–3 | display4–5 |
|----------|------------|------------|
| 状态/提示 | 目标瓶数 | 每瓶装入量 |

- `SW10=0`（编辑模式）：正在编辑的参数实时预览拨码值，以 1 Hz 闪烁；输入非法时显示默认值并闪烁，`display1` 同步闪烁作为修正提示
- `SW10=1` 但未启动：稳定显示已锁存参数

**运行态（`work_state != S_CFG`）：**

| display1 | display2–3 | display4–5 |
|----------|------------|------------|
| 空白（`4'hF`） | 已完成瓶数 | 当前瓶内药片数 |

HALT 态时全部 5 位 1 Hz 闪灭（输出 `4'hF`）。

### 闪烁机制

闪烁时钟直接使用 `clk_1Hz` 信号电平：

| 条件 | 闪烁位 |
|------|--------|
| 配置态 + 编辑模式 + `sw_sel=0` | display4–5（装入量） |
| 配置态 + 编辑模式 + `sw_sel=1` | display2–3（瓶数） |
| 配置态 + 编辑模式 + 输入非法 | display1 |
| HALT 态 | 全部 5 位 |

闪灭时输出 `4'hF`（数码管空白）。

### LG1 七段状态指示

LG1 使用 `[6:0]` 七段显示器，按优先级从高到低：

| 条件 | 显示 | 段码 |
|------|------|------|
| 配置态 + 输入非法 | E / 空白闪烁 | `7'b1111001` / `7'b0000000` |
| HALT 态 | 4 | `7'b1100110` |
| WARN 态 | 3 | `7'b1001111` |
| RUN 态 + 当前瓶即将满 | 2 | `7'b1011011` |
| RUN 态 | 1 | `7'b0000110` |
| 配置态 | 0 | `7'b0111111` |

### 实现方式

全部显示逻辑为组合逻辑（`always @(*)`），无输出寄存器，以节省 CPLD 宏单元资源。

## 9. 信号数据流

完整的控制/数据路径：

1. `pulse_sync` 用 10 kHz 采样 1 Hz，产生药片节拍脉冲 `pill_pulse`
2. `pill_ctrl_top` 对 QD 按键消抖并检测上升沿，对 `reset_sw` 做 2 级同步
3. `sw_checker` 实时检查拨码开关输入是否为合法 BCD 且在有效范围内
4. `pill_ctrl_top` 在配置态下由 QD 确认后锁存参数（非法输入锁存默认值）
5. `bottle_fsm` 根据状态机驱动计数、换瓶、报警逻辑
6. `seg_output` 根据当前状态和模式选择显示内容，控制闪烁和 LG1 状态码
7. 顶层门控蜂鸣器方波输出，区分间歇/持续两种音效
8. 5 路 BCD 码并行输出到数码管硬件

## 10. 安全机制

**按键消抖**：QD 使用 2-bit 移位寄存器检测上升沿；`reset_sw` 使用 2 级同步链。

**输入校验**：非法 BCD 值（如 `0xAB`）和越界值（如装入量 60）被拒绝，确认时锁存默认值。

**安全联锁**：运行态下 FSM 不检查 `mode_switch` 的变化——即使用户将 SW10 拨回配置位，系统也不会中断。只有 `reset_sw` 可退出运行态。

**复位扇出优化**：`sys_rst` 仅直接复位参数寄存器和 FSM 状态位；计数器通过 S_CFG 状态同步清零；`pulse_sync` 的打拍寄存器不接复位。

**FSM 兜底**：`default` 分支回到 S_CFG，防止非法状态悬空。

## 11. 硬件约束

- **器件**：EPM7128SLC84-15，128 个宏单元
- **封装**：84-pin PLCC
- **IO 标准**：TTL
- **温度范围**：0°C ~ 85°C
- **引脚使用**：37 个（详见 `pill_ctrl_top.qsf`）
- **无片上 RAM / PLL**：MAX7000S 为纯 CPLD 架构

## 13. 设计要点备注

- `acc_total`（12-bit 药片总数）在 FSM 中被计算但不在数码管上显示，运行态只显示瓶数和当前瓶药片数
- BCD 加法采用等值检测（`==9`）而非取模，适配 CPLD 乘积项架构
- 显示为 BCD 并行输出方式，非动态扫描，不需要扫描时钟
- 蜂鸣器使能信号利用状态编码的 bit[1]（`work_state[1]`），无需额外比较器
- 预判逻辑在"满瓶前一颗"时触发，实现同一时钟沿内完成换瓶
